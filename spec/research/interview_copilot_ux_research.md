# AI Interview Copilot UX 研究報告

## 1. 競品分析

### 1.1 主要產品

| 產品 | 核心功能 | 延遲 | 語言支援 |
|------|---------|------|---------|
| [LockedIn AI](https://www.lockedinai.com/) | 即時回答建議、隱形視窗 | 116ms | 42 語言 |
| [Final Round AI](https://www.finalroundai.com/interview-copilot) | 智慧/自動模式切換、個人化 | 未公開 | 91 語言 |
| [Live Interview AI](https://liveinterview.ai/) | 即時答案顯示、靜默運作 | 未公開 | 多語言 |
| [Interviews.Chat](https://www.interviews.chat) | STAR 格式回答、追問建議 | 未公開 | 多語言 |

### 1.2 關鍵 UX 設計模式

#### 顯示方式
- **獨立視窗**：建議在獨立視窗顯示，不干擾主畫面
- **Always-on-top**：保持在最上層，但可調整透明度
- **隱蔽設計**：對方看不到、不影響螢幕分享

#### 觸發模式
| 模式 | 說明 | 適用場景 |
|------|------|---------|
| **Auto Mode** | 每個偵測到的問題自動觸發建議 | 問題密集的對話 |
| **Smart Mode** | 整合連續發言，跳過非問題內容 | 自然對話流 |

#### 用戶互動
- **被動接收**：建議自動出現，用戶「像看筆記一樣自然地瞄一眼」
- **無需點擊**：不需要主動操作，降低認知負荷
- **快速鍵控制**：鍵盤快速鍵暫停/恢復，不需滑鼠

#### 個人化
- 上傳 Resume / 工作描述
- 自訂 Q&A 範例
- AI 根據背景資料生成更相關的建議

## 2. 場景對比分析

### 2.1 面試場景 vs 日常英文對話場景

| 面向 | 面試場景 | 日常英文對話場景 |
|------|---------|-----------------|
| **用戶角色** | 被動（回答問題） | 主動+被動（雙向對話） |
| **問題可預測性** | 高（面試題有套路） | 低（話題多變） |
| **思考時間** | 被允許（思考是正常的） | 較短（自然對話節奏） |
| **內容複雜度** | 中（自己的經歷） | 變化大（生活/工作問題） |
| **用戶核心需求** | 「知道怎麼答」 | 「聽懂 + 知道怎麼說」 |

### 2.2 關鍵洞察

面試助手成功的原因：
1. **用戶有明確需求**：面試是高壓力場景，用戶願意依賴 AI
2. **問題結構化**：面試問題有套路，AI 容易生成相關回答
3. **思考時間被允許**：面試中稍作思考是正常的，用戶有時間看建議
4. **單向為主**：主要是回答問題，不需要主導對話

## 3. 目標用戶場景分析

### 3.1 目標用戶
> 英文口語(會話)能力不佳的用戶，需要解決日常生活/工作的問題

### 3.2 典型場景

| 場景 | 用戶角色 | 核心痛點 | 需要的幫助 |
|------|---------|---------|-----------|
| 打電話給客服 | 主動方 | 不知道怎麼表達問題 | 講稿 + 對方回應翻譯 |
| 接到銀行/保險電話 | 被動方 | 聽不懂專業術語 | 翻譯 + 回應建議 |
| 與房東/物業溝通 | 雙向 | 表達需求 + 理解回應 | 雙向翻譯 + 建議 |
| 預約服務（醫生/維修） | 主動方 | 描述問題、確認時間 | 講稿 |
| 處理投訴/退款 | 主動方 | 據理力爭 | 講稿 + 策略建議 |
| 工作會議（多人） | 參與者 | 跟上討論 + 適時發言 | 翻譯 + 發言建議 |

### 3.3 與面試場景的相似處

1. **用戶英文能力不足**：核心痛點相同
2. **需要即時支援**：對話進行中需要幫助
3. **不想讓對方知道**：希望表現自然
4. **任務導向**：有明確目標要達成

### 3.4 與面試場景的差異

1. **用戶可能是主動方**：需要「講稿」而非只是「回答建議」
2. **雙向理解需求**：既要聽懂對方，也要表達自己
3. **場景更多樣**：不像面試有固定套路
4. **節奏更快**：日常對話不像面試可以慢慢思考

## 4. 設計建議

### 4.1 核心概念：雙向即時輔助

```
┌─────────────────────────────────────────────────────────┐
│                    英文口語輔助工具                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  對方（英語）────────────────→ 中文翻譯 ← 用戶理解       │
│                                                         │
│  用戶意圖 ────────────────────→ 英文講稿 ← 用戶照唸      │
│                                                         │
│  對方問題 ────────────────────→ 回應建議 ← 用戶參考      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 4.2 三個核心功能

#### 功能 A：即時翻譯（被動輔助）
- **觸發**：對方說話時自動觸發
- **顯示**：對方英文 → 中文翻譯
- **用途**：讓用戶理解對方在說什麼
- **UX**：持續顯示，無需操作

#### 功能 B：講稿生成（主動輔助）
- **觸發**：用戶輸入中文意圖（語音/打字）
- **顯示**：中文意圖 → 英文講稿
- **用途**：用戶照唸英文，表達自己的意思
- **UX**：用戶主動請求時生成

#### 功能 C：回應建議（情境輔助）
- **觸發**：對方問問題 / 需要回應時
- **顯示**：建議的英文回應（附中文翻譯）
- **用途**：用戶不知道怎麼回應時參考
- **UX**：Smart Mode，只在需要時顯示

### 4.3 UI 設計建議

```
┌──────────────────────────────────────────────────────────┐
│ 🎧 英文對話助手                            [暫停] [設定] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 🔊 對方說：                                         │ │
│  │ "Hi, I'm calling about your recent order.          │ │
│  │  There seems to be an issue with the delivery."    │ │
│  │                                                    │ │
│  │ 📝 翻譯：                                          │ │
│  │ 「嗨，我是來電關於你最近的訂單。                     │ │
│  │   配送似乎有問題。」                                │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 💬 建議回應：                                       │ │
│  │                                                    │ │
│  │ "Oh, I see. What's the issue with the delivery?"  │ │
│  │ （哦，我了解。配送有什麼問題？）                     │ │
│  │                                                    │ │
│  │ [照唸] [換一個]                                    │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  ┌────────────────────────────────────────────────────┐ │
│  │ ✏️ 我想說：[輸入中文，生成英文講稿...]              │ │

│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 4.4 與面試助手的差異

| 面向 | 面試助手 | 本工具 |
|------|---------|-------|
| 主要功能 | 回答建議 | 翻譯 + 講稿 + 建議 |
| 用戶輸入 | 被動（聽問題） | 主動+被動（可輸入意圖） |
| 建議觸發 | 每個問題 | 智慧判斷（需要時才顯示） |
| 個人化 | Resume + JD | 場景設定 + 目標 |

### 4.5 建議的模式切換

| 模式 | 說明 | 適用場景 |
|------|------|---------|
| **傾聽模式** | 翻譯為主，建議為輔 | 聽對方說明、接收資訊 |
| **對話模式** | 翻譯 + 建議 + 講稿 | 雙向溝通 |
| **發言模式** | 講稿生成為主 | 用戶需要表達較長內容 |

## 5. 與原 pivot_design_v2.md 的調整建議

### 5.1 保留的設計
- Mode A / Mode B 雙模式架構
- 即時翻譯功能
- 筆記功能

### 5.2 調整的設計

| 原設計 | 調整建議 | 原因 |
|-------|---------|------|
| 「建議回應」每句話觸發 | 改為 Smart Mode，問題/需回應時才觸發 | 降低干擾 |
| 建議只有英文 | 建議附中文翻譯 | 用戶需要理解建議內容 |
| 無「講稿生成」功能 | 新增：用戶輸入中文 → 生成英文講稿 | 用戶可能是主動方 |
| 喚醒後 AI 直接說 | 改為：顯示講稿，用戶自己唸 | 更自然、用戶保持控制 |

### 5.3 新增的設計

1. **講稿輸入框**：用戶輸入中文意圖，生成英文講稿
2. **建議附翻譯**：英文建議 + 中文翻譯
3. **模式切換**：傾聽/對話/發言模式
