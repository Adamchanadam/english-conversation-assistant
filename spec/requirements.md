# English Conversation Assistant — Requirements v2.0

## 1. 產品背景與定位

本產品為「英文對話助手」（English Conversation Assistant, ECA）：協助非英語母語用戶在英文電話中**聽懂對方**並**知道怎麼說**。

**核心理念**：用戶主導對話，AI 提供即時輔助。

**目標用戶**：香港 BNO 移民在英國（150,400+ 人），需要打電話給 GP/銀行/Council 等機構。

**核心痛點**：
- 66% 非母語者有電話焦慮
- 聽不懂對方說什麼
- 不知道自己該怎麼回應

## 2. 核心目標（Product Goal）

讓用戶在英文電話中：
1. **即時理解**對方說的英文（英→中翻譯）
2. **知道怎麼說**自己想表達的內容（中→英講稿）
3. **不會卡住**（Panic Button + 快捷短語）

## 3. 模型與工具限制（Hard Constraints）

僅允許使用以下 OpenAI 模型（硬性規則，不可變更）：

| 模型 | 用途 | 限制 |
|------|------|------|
| `gpt-realtime-mini` | 即時語音轉錄（STT） | 32k context，不支援 Structured Outputs |
| `gpt-5-mini` | 講稿生成 + Smart 建議 | 400k context，128k max output |
| `gpt-4.1-nano` | 即時翻譯（英→中） | 最快模型，703ms 首字回應 |

## 4. 主要使用情境（Primary Use Cases）

| ID | 情境 | 說明 |
|----|------|------|
| UC-01 | 即時翻譯 | 對方說英文 → 用戶看到中文字幕 |
| UC-02 | 講稿生成 | 用戶輸入中文 → 生成英文講稿供用戶照唸 |
| UC-03 | Smart 建議 | 對方問問題 → AI 建議幾個回應選項 |
| UC-04 | Panic Button | 用戶卡住 → 按鈕讓 AI 說「讓我想想」爭取時間 |
| UC-05 | 快捷短語 | 常用語一鍵使用（請再說一次、請慢點說等） |

## 5. 功能需求（Functional Requirements）

### 5.1 即時翻譯（P0 - MVP 必須）

**功能**：對方說英文 → 即時顯示中文字幕

**輸入來源**：
- 麥克風模式（推薦）：用戶開擴音，麥克風捕獲對方聲音
- 系統音訊模式（進階）：`getDisplayMedia()` 捕獲系統音訊

**輸出**：
- 英文原文（即時串流顯示）
- 中文翻譯（即時串流顯示）

**性能要求**：
| 指標 | 目標 | 說明 |
|------|------|------|
| 英文即時顯示 | < 100ms | Web Speech API 本地處理 |
| 分段延遲 | < 800ms | 從停止說話到觸發分段 |
| 翻譯開始延遲 | < 500ms | 句子結束後開始翻譯 |
| 翻譯端到端延遲 | < 1.5s | 從說完到翻譯顯示完成 |
| 準確度 | > 85% 用戶滿意 | 人工測試 |

**智能分段要求**：
- 系統必須能在對方連續說話時智能識別句子邊界
- 不等整段說完就開始翻譯（邊說邊翻）
- 每個分段獨立處理，新分段不阻塞舊分段的翻譯

**Entry 管理要求**：
- 每段對話（由 OpenAI `item_id` 定義）為一個 Entry
- 每個 Entry 包含：英文原文 + 中文翻譯
- 不能有孤兒 Entry（有英文無翻譯，或有翻譯無英文）
- 不能有內容錯配（A 的英文配 B 的翻譯）

### 5.2 講稿生成（P0 - MVP 必須）

**功能**：用戶輸入中文 → 生成英文講稿

**輸入**：
- 中文文字輸入
- 場景選擇（bank/nhs/utilities 等）
- 對話上下文

**輸出**：
- 英文講稿（主要）
- 替代說法（2-3 個選項）
- 發音提示（可選）

**性能要求**：
| 指標 | 目標 |
|------|------|
| 生成延遲 | < 1.5s |
| 串流輸出 | 首字 < 500ms |

### 5.3 Panic Button（P0 - MVP 必須）

**功能**：用戶卡住時，按鈕觸發「買時間」短語 + 生成建議

**行為**：
1. 按下按鈕 → 立即顯示/播放「讓我想想...」短語
2. 同時開始生成回應建議
3. 顯示 3 個建議選項供用戶選擇

**性能要求**：
| 指標 | 目標 |
|------|------|
| 短語顯示 | < 300ms |
| 建議生成 | < 1.5s |

### 5.4 快捷短語（P1 - MVP 必須）

**功能**：常用語一鍵使用

**預設短語**：
| 中文 | 英文 |
|------|------|
| 請再說一次 | Could you repeat that, please? |
| 請慢點說 | Could you speak more slowly? |
| 我確認一下 | Let me confirm that... |
| 謝謝再見 | Thank you. Goodbye. |

### 5.5 Smart 建議（P1 - v1.5）

**功能**：根據對話上下文，智能顯示回應建議

**觸發條件**：
- 對方問問題（問號、疑問詞）
- 沉默超過 3 秒
- 等待確認的語句

**輸出**：
- 最多 3 個建議
- 每個建議包含：英文 + 中文翻譯
- 5 秒後自動隱藏（非侵入式）

### 5.6 場景預設（P2 - v1.5）

**功能**：預設常見場景的詞彙和建議

**核心場景**：
- 銀行/金融
- NHS/GP
- 公用事業
- 保險

## 6. 非功能需求（NFR）

### 6.1 延遲

| 功能 | 目標延遲 |
|------|---------|
| 翻譯顯示 | < 500ms |
| 講稿生成 | < 1.5s |
| Panic Button | < 300ms |

### 6.2 可用性

- 手機版響應式設計
- 暗色主題（減少眼睛疲勞）
- 大字體講稿顯示（Teleprompter 樣式）

### 6.3 Session 限制

- 單一 Realtime session 最長 60 分鐘
- 需設計自動重連機制

### 6.4 成本控制

- 記錄 API 用量
- 設置預算警戒線

## 7. 成功準則（Success Criteria）

### MVP 驗收標準

| 指標 | 目標 | 驗收方式 |
|------|------|---------|
| 翻譯準確度 | > 85% 用戶滿意 | 人工測試 10 句 |
| 翻譯延遲 | < 500ms | 計時測量 |
| 講稿生成延遲 | < 1.5s | 計時測量 |
| Panic Button 響應 | < 300ms | 計時測量 |
| 手機版可用性 | 核心功能正常 | 手動測試 |

### v1.0 指標

| 指標 | 目標 |
|------|------|
| 翻譯準確度 | > 90% 用戶滿意 |
| 講稿採用率 | > 60% |
| NPS | > 50 |

## 8. 風險與限制

| 風險 | 影響 | 緩解措施 |
|------|------|---------|
| 翻譯準確度不足 | 用戶誤解 | 顯示原文 + 信心指示 |
| 延遲過高 | UX 差 | 多重音訊來源備援 |
| 系統音訊捕獲限制 | 部分系統不支援 | 麥克風模式作為主要方案 |
| 網路不穩定 | 連線中斷 | 自動重連 + 離線提示 |
